# Demo Video Script - FRMS

## Total Duration: 6-8 Minutes

## 0:00 - 0:30 | Introduction
"Hello everyone, this is our project Faculty Research Management System, or FRMS. It is designed to manage faculty research records, approvals, reports, and AI-assisted analytics in one platform."

## 0:30 - 1:10 | Problem and Need
"Institutions usually handle publications and projects using manual spreadsheets and email approvals. This causes delays, missing records, and difficulty in generating NAAC/NBA/NIRF reports."

## 1:10 - 1:50 | Architecture Overview
"Our frontend is built in React and deployed on Vercel. Backend is Node.js and Express on Render. Data is stored in MongoDB. APIs are documented through Swagger."

## 1:50 - 2:40 | Login and Role-Based Access
1. Show login screen.
2. Login as Super Admin.
3. Show dashboard and role-specific sidebar.
4. Mention that each role sees only allowed modules.

Narration:
"FRMS uses JWT authentication and role-based access control for secure module access."

## 2:40 - 3:40 | Research Module Workflow
1. Open Publications module.
2. Add a publication entry.
3. Show pending status.
4. Switch to reviewer/admin role and approve entry.

Narration:
"Each entry follows approval workflow, and complete status history is tracked."

## 3:40 - 4:30 | Reports
1. Open Reports page.
2. Generate one report (for example faculty-wise PDF).
3. Open generated report link.

Narration:
"Reports are generated in PDF or Excel format and can be used directly for accreditation and review."

## 4:30 - 5:30 | AI Module
1. Open AI module.
2. Show one or two tools:
   - Research summary
   - Publication recommendation
   - Smart search

Narration:
"AI features reduce manual effort in analysis, recommendation, and documentation support."

## 5:30 - 6:10 | Notifications and Audit Logs
1. Open Notifications page.
2. Open Audit Logs page.

Narration:
"The system provides transparency with notification tracking and admin audit logs."

## 6:10 - 6:40 | Mobile + UX Enhancements
1. Show responsive layout on narrow viewport.
2. Show dark mode toggle.
3. Trigger one toast notification.

Narration:
"The UI is optimized for mobile and desktop, with dark mode, loading skeletons, and clear feedback notifications."

## 6:40 - 7:00 | Closing
"FRMS offers a complete, secure, and scalable platform for research lifecycle management. Thank you."
