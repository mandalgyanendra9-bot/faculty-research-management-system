# User Manual - FRMS

## 1. Login and Registration
- Open landing page and click `Login` or `Register`.
- Public registration creates a pending faculty account.
- After admin approval, login is enabled.

## 2. Faculty Workflow
- Submit research entries under Publications/Projects/Patents/Grants/Events.
- Track approval status and notifications.
- Use AI Suite tabs for summary, recommendations, search, plagiarism upload, proposal drafting, CV, chat, and OCR.
- Generate personal reports from Reports page.

## 3. Admin/Super Admin Workflow
- Open `Users` page to:
  - Approve pending faculty.
  - Assign roles (`faculty`, `hod_dean`, `research_coordinator`).
  - Manage departments and lookups.
  - Review recent role-change audit logs.
- Use `Approvals` for record moderation.
- Use `Reports` for NAAC/NBA/NIRF/institutional exports.
- Use `Audit Logs` for compliance monitoring.

## 4. Reports
- Select report type and format (`PDF` or `Excel`).
- Click generate and open the downloadable output link.

## 5. Troubleshooting
- If API calls are slow initially, wait for Render cold start.
- If register/login fails, verify account activation and email correctness.
- If dropdowns are empty, ensure departments/lookups exist (defaults auto-seeded by backend lookup endpoints).
